## 1. 快速结论

### 1.1 AIS 是什么

AIS，Automatic Identification System，船舶自动识别系统，是基于 VHF 海上移动通信频段的数字广播系统。它让船舶、岸基站、航标、搜救设备等周期性发送身份、位置、航速、航向、航次、状态等信息，用于船舶避碰、海事监管、船舶交通服务、港口调度、搜救和航运数据分析。

### 1.2 DSC 是什么

DSC，Digital Selective Calling，数字选择性呼叫，是 GMDSS 全球海上遇险与安全系统的重要组成部分。它用于在 VHF/MF/HF 无线电上自动发送数字呼叫，完成遇险报警、紧急呼叫、安全呼叫、单船呼叫、群呼、区域呼叫、电话/无线电通信建立等功能。

### 1.3 二者的核心区别

| 维度 | AIS | DSC |
|---|---|---|
| 核心目标 | 自动广播识别与动态态势感知 | 数字选择性呼叫与遇险安全通信建立 |
| 通信模式 | 以广播为主，周期性自动发送 | 以呼叫为主，事件触发或人工触发 |
| 典型信息 | MMSI、船名、位置、航速、航向、航行状态、目的港 | 遇险类型、MMSI、位置、时间、呼叫类别、后续通信信道 |
| 典型频率 | VHF AIS 1 / AIS 2，常见为 161.975 MHz / 162.025 MHz | VHF DSC Ch70，156.525 MHz；MF/HF 也有 DSC 频率 |
| 典型调制 | GMSK，9600 bps | VHF DSC 常用 1200 bps FSK；MF/HF DSC 常用窄带 FSK |
| 典型设备 | 船载 AIS、AIS 基站、AIS AtoN、AIS SART、AIS 接收机 | VHF DSC 电台、MF/HF DSC 电台、GMDSS 控制台 |
| 实时态势能力 | 强 | 弱 |
| 遇险报警法定体系 | 辅助搜救与定位 | GMDSS 核心报警手段之一 |

## 2. 海上通信背景

### 2.1 VHF 海上通信

VHF 海上通信主要工作在 156 MHz 至 174 MHz 附近的海上移动业务频段。其特点是视距传播、低时延、设备成熟、成本较低、船岸和船船通信广泛使用。

影响通信距离的关键因素：

- 天线高度：VHF 主要为视距传播，天线越高，覆盖越远。
- 发射功率：船载 VHF 通常 1 W/25 W 可切换，AIS Class A 常见 12.5 W，Class B 常见 2 W 或 5 W。
- 接收机灵敏度：灵敏度越高，弱信号解码能力越好。
- 天线增益与馈线损耗：高增益天线和低损耗馈线可以显著改善链路预算。
- 电磁环境：港口、船厂、城市岸线、基站密集区可能存在同频/邻频干扰。
- 地形遮挡：岛屿、山体、建筑物会影响覆盖。

### 2.2 GMDSS 与海上安全通信

GMDSS，Global Maritime Distress and Safety System，是国际海事组织推动的全球海上遇险与安全系统。它综合使用 DSC、NAVTEX、Inmarsat、EPIRB、SART、VHF/MF/HF 等手段，实现遇险报警、搜救协调、安全信息播发。

DSC 是 GMDSS 中非常关键的数字报警与呼叫机制，尤其是 VHF DSC Channel 70。AIS 虽然也用于搜救和态势感知，但 AIS 不是 DSC 的替代品。

### 2.3 MMSI

MMSI，Maritime Mobile Service Identity，海上移动业务识别码，是船舶电台、岸台、群组、搜救设备等的数字身份。

常见用途：

- AIS 报文中的船舶身份。
- DSC 呼叫中的目标身份与发起方身份。
- VHF/MF/HF 电台、船站、岸站、群呼地址识别。

工程注意：MMSI 不是随意编造的字段。实网发射中冒用、伪造 MMSI 可能造成监管、安全和法律风险。

## 3. AIS 原理详解

### 3.1 AIS 的系统目标

AIS 设计目标是让参与海上交通的目标自动交换关键信息，降低碰撞风险，提高水域管理效率。

典型目标包括：

- 船舶之间自动互见。
- 岸基 VTS 或海事监管系统实时掌握船舶动态。
- 航标向船舶广播自身位置与状态。
- 搜救设备向附近船舶和岸站广播落水或遇险位置。
- 航运数据平台采集船舶轨迹，形成港口、船队、航线、ETA 等数据服务。

### 3.2 AIS 的网络形态

AIS 是一种自组织、时分复用的 VHF 数字通信系统。

主要节点：

- 船载 AIS Class A：SOLAS 船舶常用，发送功率高、发送频率高、功能完整。
- 船载 AIS Class B：小型船、游艇、非强制船舶常用，成本较低，发送频率较低。
- AIS 基站/Base Station：岸基或平台站，用于管理区域内 AIS 信道、播发基站信息、接收船舶信息。
- AIS AtoN：Aid to Navigation，航标 AIS，可代表实体航标、合成航标或虚拟航标。
- AIS SART：Search and Rescue Transmitter，搜救定位发射设备。
- AIS 接收机：只接收，不发射，用于数据采集、监测、展示。

### 3.3 AIS 频率与信道

常见 AIS 信道：

| 名称 | 频率 | 说明 |
|---|---:|---|
| AIS 1 | 161.975 MHz | VHF Channel 87B，主 AIS 信道之一 |
| AIS 2 | 162.025 MHz | VHF Channel 88B，主 AIS 信道之一 |

其他相关：

- DSC VHF Channel 70：156.525 MHz，不是 AIS 信道。
- VHF 语音 Channel 16：156.800 MHz，国际遇险、安全和呼叫频道。

工程注意：频率公开不代表可以自由发射。接收、研发、屏蔽测试、实网发射、岸基台站运营在法规上是不同问题。

### 3.4 AIS 物理层

典型参数：

| 参数 | 典型值 |
|---|---|
| 调制方式 | GMSK |
| 速率 | 9600 bit/s |
| 信道带宽 | 25 kHz |
| 接入方式 | TDMA，包含 SOTDMA、CSTDMA、FATDMA、RATDMA 等机制 |
| 单时隙长度 | 约 26.67 ms |
| 每分钟时隙数 | 2250 slots/channel |
| 双信道总时隙 | 约 4500 slots/min |
| 发射功率 | Class A 常见 12.5 W；Class B 常见 2 W 或 5 W，具体取决于类别和设备 |

### 3.5 AIS 链路层与 TDMA

AIS 的核心难点不是单个报文调制，而是多个船舶在同一区域内共享有限时隙。

常见接入方式：

- SOTDMA，Self-Organized TDMA：Class A 常用。设备根据周围时隙占用情况自主预约未来时隙。
- CSTDMA，Carrier Sense TDMA：部分 Class B 常用。发送前监听信道，尽量选择空闲时隙。
- FATDMA，Fixed Access TDMA：固定分配时隙，多用于基站、航标等受控节点。
- RATDMA，Random Access TDMA：随机接入，常用于初始接入或特殊消息。

### 3.6 AIS 报文结构

AIS 报文由底层比特流承载，上层常以 NMEA 0183 的 AIVDM/AIVDO 句子输出给软件系统。

常见 NMEA 句子：

```text
!AIVDM,1,1,,A,15Muq?002>G?svP00<:O?vN60<0,0*5C
!AIVDO,1,1,,,B3P;Q@P00?wPabP?wvN03wwP0000,0*31
```

字段含义概念：

- `!AIVDM`：接收到的其他目标 AIS 信息。
- `!AIVDO`：本船自身 AIS 信息。
- 分片数量/序号：长报文可能拆分。
- 信道标识：A/B，对应 AIS 1/AIS 2。
- Payload：6-bit ASCII 编码后的 AIS 二进制载荷。
- Fill bits：填充位数量。
- Checksum：NMEA 校验。

### 3.7 常见 AIS 消息类型

| 消息类型 | 名称 | 主要内容 | 场景 |
|---:|---|---|---|
| 1/2/3 | Class A 位置报告 | MMSI、经纬度、SOG、COG、航向、转向率、航行状态 | 大型船舶动态跟踪 |
| 4 | 基站报告 | 基站位置、UTC 时间 | 岸基基站、时间同步 |
| 5 | Class A 静态与航次数据 | 船名、呼号、IMO、船型、尺寸、目的港、ETA | 船舶身份与航次信息 |
| 6 | 寻址二进制消息 | 目标 MMSI、应用数据 | 定向数据业务 |
| 8 | 广播二进制消息 | 应用数据 | 气象、水文、区域信息等 |
| 9 | SAR 航空器位置 | 搜救航空器位置 | 搜救 |
| 12 | 寻址安全相关文本 | 安全短消息 | 安全通信辅助 |
| 14 | 广播安全相关文本 | 安全广播 | 航行警告辅助 |
| 18 | Class B 标准位置报告 | MMSI、经纬度、SOG、COG | 小型船舶动态 |
| 19 | Class B 扩展位置报告 | 动态 + 部分静态信息 | Class B 扩展上报 |
| 21 | AtoN 报告 | 航标类型、位置、状态 | 实体/虚拟航标 |
| 24 | Class B 静态数据 | 船名、船型、尺寸等 | 小型船舶身份 |
| 27 | 长距离 AIS 广播 | 简化位置，鲁棒性更强 | 远距离/卫星 AIS |

### 3.8 AIS 更新周期

AIS 发送周期取决于船舶类别、航速、转向状态、锚泊状态和消息类型。

典型规律：

- Class A 高速或转向时更新更频繁，可达到数秒级。
- Class A 锚泊或低速时更新较慢。
- Class B 通常更新频率低于 Class A。
- 静态航次类消息如 Type 5、Type 24 更新周期通常为分钟级。

工程意义：

- 做轨迹平滑时不能假设所有船舶固定 1 秒或 10 秒上报。
- 做离线分析时需处理时序不均匀、丢包、重复包、跳点。
- 做实时告警时应结合目标类型、速度、最后更新时间设定不同超时阈值。

### 3.9 AIS 接收链路

典型接收链路：

```text
VHF 天线 -> 避雷/滤波 -> 低损耗馈线 -> AIS 接收机/SDR -> 解调 -> AIS 二进制解码 -> NMEA/JSON -> 存储/展示/API
```

关键工程指标：

- 天线中心频率：覆盖 162 MHz 附近。
- 天线类型：船用 VHF 天线、岸基高增益 VHF 天线、宽带接收天线。
- 馈线：RG-58 损耗较大，长距离建议更低损耗馈线。
- 接收机灵敏度：影响弱信号接收。
- 动态范围：港口强信号环境下防止前端过载。
- 双通道能力：理想接收机可同时接收 AIS 1 和 AIS 2。
- 时间戳精度：多站 TDOA 或高精度轨迹分析需要更准时间戳。

### 3.10 AIS 数据处理链路

典型软件处理流程：

```text
NMEA 输入 -> 校验 -> 分片重组 -> 6-bit 解码 -> 消息类型解析 -> 坐标/字段归一化 -> 质量控制 -> 轨迹入库 -> 实时推送/地图展示/告警分析
```

关键软件模块：

- NMEA 采集：串口、TCP、UDP、文件流、消息队列。
- 校验与分片：NMEA checksum、multipart 组包超时。
- AIS 解码：按消息类型解析位字段。
- 坐标处理：经纬度单位换算、非法值过滤。
- 轨迹清洗：去重、乱序处理、跳点检测、速度合理性检查。
- 目标融合：多岸站同一 MMSI 数据合并。
- 船舶档案：MMSI、IMO、船名、船型、尺寸、呼号映射。
- 地图服务：WebSocket、矢量瓦片、轨迹回放。
- 告警引擎：禁区闯入、偏航、靠离泊、CPA/TCPA 碰撞风险。
- 数据安全：脱敏、权限控制、审计、跨境控制。

## 4. DSC 原理详解

### 4.1 DSC 的系统目标

DSC 的核心目标是“先用数字信令把目标叫起来，再切换到指定信道进行语音或后续通信”。它解决传统语音呼叫中人工守听效率低、遇险信息不结构化、目标选择不精确的问题。

DSC 可实现：

- 一键遇险报警。
- 船对岸、岸对船、船对船选择性呼叫。
- 群呼、区域呼叫。
- 自动携带 MMSI、位置、时间、遇险类型等结构化信息。
- 指定后续通信方式和信道。

### 4.2 DSC 与 GMDSS

DSC 是 GMDSS 的组成部分。VHF DSC 主要用于近海 A1 海区；MF/HF DSC 用于更远距离海区。

常见 GMDSS 海区：

| 海区 | 概念 | 典型通信能力 |
|---|---|---|
| A1 | 至少一个 VHF 岸台 DSC 覆盖范围 | VHF DSC |
| A2 | A1 之外，至少一个 MF 岸台 DSC 覆盖范围 | MF DSC |
| A3 | A1/A2 外，Inmarsat 覆盖范围 | 卫星 + HF/MF |
| A4 | A1/A2/A3 外，极区等 | HF DSC 等 |

### 4.3 DSC 频率

常见 DSC 频率：

| 类型 | 频率 | 用途 |
|---|---:|---|
| VHF DSC Ch70 | 156.525 MHz | VHF 数字选择性呼叫，不用于普通语音 |
| MF DSC | 2187.5 kHz | 中频遇险与安全 DSC |
| HF DSC | 4207.5 / 6312 / 8414.5 / 12577 / 16804.5 kHz 等 | 高频远距离 DSC |

工程注意：VHF DSC Channel 70 是专用 DSC 信道，普通语音通信应切换到指定工作信道，不应在 Ch70 上语音占用。

### 4.4 DSC 物理层

VHF DSC 典型参数：

| 参数 | 典型值 |
|---|---|
| 频率 | 156.525 MHz |
| 调制 | FSK |
| 速率 | 1200 bit/s |
| 频移 | 约 1300/2100 Hz 音频 FSK 概念常见于实现描述 |
| 信道 | VHF Channel 70 |
| 典型用途 | 遇险报警、单呼、群呼、岸台呼叫、后续信道协商 |

MF/HF DSC 典型特征：

- 使用窄带 FSK。
- 速率通常低于 VHF DSC。
- 面向远距离传播，受电离层、时间、季节、频段影响明显。

### 4.5 DSC 呼叫类型

常见呼叫类别：

- Distress：遇险。
- Urgency：紧急。
- Safety：安全。
- Routine：常规业务。

常见目标类型：

- 单船呼叫。
- 岸台呼叫。
- 群呼。
- 区域呼叫。
- 全船呼叫。

### 4.6 DSC 遇险报警内容

典型遇险报警包含：

- 本台 MMSI。
- 遇险性质：火灾、进水、碰撞、搁浅、倾覆、失控、弃船、人员落水、海盗等。
- 位置：来自 GNSS 或人工输入。
- 时间：UTC。
- 后续通信方式：通常切换到 VHF Ch16 或指定频率。

工程注意：DSC 电台应连接 GNSS 或具备可靠位置输入，否则遇险报警可能缺少位置，降低搜救效率。

### 4.7 DSC 工作流程示例

遇险报警：

```text
船舶按下 DSC Distress -> 电台在 DSC 信道发送遇险数字报警 -> 岸台/附近船舶收到并告警 -> 后续切换到语音遇险信道 -> 搜救协调
```

常规单船呼叫：

```text
输入目标 MMSI -> 选择 Routine 呼叫 -> 指定后续工作信道 -> 发送 DSC 呼叫 -> 对方确认 -> 双方切换到工作信道语音通信
```

## 5. AIS 与 DSC 应用场景对比

### 5.1 船舶避碰

AIS：强相关。AIS 提供附近船舶位置、速度、航向，可辅助 ECDIS、雷达、ARPA 进行态势感知。  
DSC：弱相关。DSC 可用于联系目标船，但不提供连续态势。

### 5.2 海事监管/VTS

AIS：核心数据源之一，用于船舶流量监管、禁限航区监控、靠离泊识别、异常行为发现。  
DSC：用于通信调度、选择性呼叫、遇险与安全通信。

### 5.3 遇险搜救

AIS：AIS SART、AIS MOB、AIS 目标可提供位置广播，便于附近船舶和岸站发现。  
DSC：遇险报警体系核心之一，可结构化发送遇险信息并触发岸台/船台告警。

### 5.4 港口与航运运营

AIS：可用于 ETA 预测、泊位调度、船队管理、拥堵分析、历史轨迹回放。  
DSC：更多用于通信联系，不适合作为运营态势主数据源。

### 5.5 航标与水上施工

AIS：AIS AtoN 可表达实体航标、虚拟航标、施工区、危险区提示。  
DSC：可发布安全呼叫或联系特定船舶，但不是航标展示系统。

### 5.6 数据产品与商业服务

AIS：非常适合做数据产品，包括船舶档案、轨迹、港口靠泊、航线、运力、贸易流分析。  
DSC：数据敏感、业务专用、安全属性强，通常不作为公开商业数据源。

## 6. 硬件规格与工程参数

### 6.1 AIS 船载 Class A 设备

典型规格：

| 项目 | 典型要求/范围 |
|---|---|
| 工作频率 | AIS 1 161.975 MHz，AIS 2 162.025 MHz |
| 发射功率 | 常见 12.5 W，可按设备/标准支持低功率模式 |
| 调制方式 | GMSK |
| 数据速率 | 9600 bps |
| GNSS | 内置或外接 GNSS，用于位置和时间 |
| 接口 | NMEA 0183、NMEA 2000、RS-422/RS-232、以太网，视设备而定 |
| 显示 | MKD，Minimum Keyboard and Display，或集成显示 |
| 输入 | 航向、转向率、导航状态、目的港、ETA 等 |
| 输出 | AIS NMEA、报警、状态、目标列表 |
| 供电 | 船舶 DC 12/24 V 常见 |

工程关注：

- GNSS 天线与 VHF 天线隔离。
- 与 ECDIS、雷达、VDR、综合导航系统接口兼容。
- 静态信息配置准确性：MMSI、船名、呼号、尺寸、天线偏移。
- 船舶尺寸与 GNSS 天线偏移错误会导致地图目标位置偏移。

### 6.2 AIS Class B 设备

典型规格：

| 项目 | Class B 常见特征 |
|---|---|
| 发射功率 | 2 W 或 5 W，取决于 Class B 类型 |
| 接入方式 | CSTDMA 或 SOTDMA，取决于设备类别 |
| 更新频率 | 通常低于 Class A |
| 目标用户 | 小型商船、渔船、游艇、工作艇 |
| 成本 | 通常低于 Class A |

工程关注：

- 高密度水域中 Class B 报文可能更容易被 Class A/基站流量挤压。
- 小船电源、天线高度和安装质量对效果影响很大。

### 6.3 AIS 接收机/岸基接收站

典型规格建议：

| 项目 | 建议 |
|---|---|
| 接收通道 | 双通道同时接收 AIS 1/AIS 2 优先 |
| 灵敏度 | 越低越好，工程上关注真实解码率而非纸面数值 |
| 动态范围 | 港口强信号环境下非常重要 |
| 输出接口 | USB、串口、TCP、UDP、MQTT、HTTP API |
| 输出格式 | NMEA 0183 AIVDM/AIVDO，或 JSON |
| 时间同步 | NTP/PTP/GNSS 时间戳，视定位和融合需求 |
| 供电 | DC、PoE、UPS 备电 |
| 防护 | 户外 IP65/IP67 箱体、防雷、接地、防盐雾 |

岸基站点设计：

- 天线尽量高且视野开阔。
- 采用低损耗馈线，减少长线损耗。
- 增加带通滤波器抑制强干扰。
- 做好避雷、接地、防水、防盐雾。
- 网络链路建议有 4G/5G/有线双链路冗余。

### 6.4 AIS 岸基发射/基站设备

典型能力：

- 接收 AIS 1/AIS 2。
- 发射基站报告或管理消息。
- 支持 FATDMA/RATDMA 等时隙管理机制。
- 与 VTS、海事监管平台、航标管理系统连接。
- 支持远程配置、日志、告警、状态监测。

合规与工程注意：

- 需关注无线电频率许可、台站执照、设备型号核准、台址审批、海事主管要求。
- 不建议企业自行实网发射测试。
- 实验室测试应使用屏蔽箱和假负载。

### 6.5 DSC VHF 电台

典型规格：

| 项目 | 典型值/说明 |
|---|---|
| DSC 信道 | VHF Ch70，156.525 MHz |
| 语音信道 | Ch16 156.800 MHz 及其他工作信道 |
| 发射功率 | 常见 1 W/25 W 可切换 |
| DSC 调制 | FSK，VHF 常见 1200 bps |
| 身份 | MMSI |
| GNSS | 内置或外接，用于遇险位置 |
| 接口 | NMEA 0183/2000、GNSS 输入、外接报警、扬声器 |
| 功能 | Distress、All Ships、Individual、Group、Position Request 等 |

工程关注：

- DSC 电台必须正确配置 MMSI。
- GNSS 位置输入要稳定。
- 遇险按钮需要防误触设计。
- 电台与天线安装质量影响语音和 DSC 成功率。

### 6.6 MF/HF DSC 电台

典型特征：

- 覆盖远距离通信。
- 天线系统更复杂，需要调谐器、良好接地和反射网络。
- 传播受频段、昼夜、电离层、太阳活动影响。
- 设备成本、安装调试难度高于 VHF。

## 7. 软件规格与数据接口

### 7.1 AIS 常见输入输出格式

常见输入：

- 串口 NMEA 0183。
- TCP Server/Client NMEA 流。
- UDP NMEA 广播。
- 文件回放。
- SDR IQ 数据。
- 厂商私有 API。

常见输出：

- 原始 NMEA。
- 解析后 JSON。
- Protobuf/Avro 等二进制消息。
- WebSocket 实时目标流。
- REST API 查询。
- 时序数据库/空间数据库。

JSON 示例：

```json
{
  "message_type": 1,
  "mmsi": 413123456,
  "timestamp": "2026-05-19T10:20:30Z",
  "lat": 31.2304,
  "lon": 121.4737,
  "sog": 12.4,
  "cog": 87.5,
  "heading": 89,
  "nav_status": "under_way_using_engine",
  "source_station": "SHA-001",
  "channel": "A"
}
```

### 7.2 AIS 数据库设计建议

核心表：

- vessel_static：船舶静态档案，MMSI、IMO、船名、呼号、船型、尺寸。
- vessel_position_latest：最新位置。
- vessel_position_history：历史轨迹。
- raw_ais_nmea：原始 NMEA 留存。
- station_status：接收站状态。
- data_quality_event：质量问题记录。
- geofence_event：电子围栏事件。

关键索引：

- MMSI + 时间。
- 空间索引，经纬度或 geometry/geography。
- station_id + 时间。
- message_type + 时间。

### 7.3 AIS 数据质量控制

常见问题：

- MMSI 异常或重复。
- 经纬度为非法值或 0 点。
- 航速异常，如瞬时跳变到不合理值。
- 船名乱码或填充字符。
- 静态信息长期错误。
- 同一报文多站重复接收。
- 乱序到达。
- 分片丢失。

建议策略：

- 原始数据完整留存，解析数据可清洗。
- 对位置、速度、航向设置合理阈值。
- 使用 H3/GeoHash/空间索引提升查询效率。
- 多站融合时保留 source_station 和 RSSI/信号质量。
- 对敏感目标设置脱敏与访问控制。

### 7.4 DSC 软件集成

DSC 数据通常来自专业电台或 GMDSS 控制台，不像 AIS 一样大量开放输出。工程集成重点：

- 电台事件日志采集。
- 遇险报警记录。
- MMSI 与船舶档案关联。
- 呼叫类别、目标、时间、位置、后续信道记录。
- 与值班系统、告警系统、录音系统联动。

### 7.5 告警与业务算法

AIS 常见算法：

- CPA/TCPA：最近会遇距离/时间。
- Geofence：禁区、锚地、港区、施工区进出。
- Port Call：靠港、离港、泊位占用识别。
- ETA：到港时间预测。
- Anomaly：异常停留、异常转向、AIS 关闭/消失、速度异常。
- Route Deviation：偏航。
- Fleet Monitoring：船队监控。

DSC 常见联动：

- 遇险报警弹窗。
- 自动定位到报警位置。
- 关联附近 AIS 目标。
- 自动生成事件单。
- 录音与通信记录归档。

## 10. 销售沟通要点

### 10.1 AIS 的销售语言

AIS 是“船舶位置与航行状态的自动广播数据源”。它解决的是“看得见、管得住、算得清”的问题。

客户价值：

- 看得见：实时掌握船舶位置、航速、航向、身份。
- 管得住：电子围栏、异常行为、禁区闯入、靠离泊监管。
- 算得清：港口吞吐、船舶周转、航线效率、拥堵分析。

### 10.2 DSC 的销售语言

DSC 是“海上安全通信的数字呼叫与遇险报警机制”。它解决的是“叫得到、叫得准、报警快”的问题。

客户价值：

- 叫得到：用 MMSI 精确呼叫指定船舶或群组。
- 叫得准：数字呼叫携带身份、类别、位置、后续信道。
- 报警快：遇险时一键发送结构化报警，提高应急响应速度。

### 10.3 AIS 与 DSC 的组合价值

组合后可以实现：

- 地图上发现目标。
- 用 DSC 精确呼叫目标。
- 遇险报警自动关联 AIS 位置与附近船舶。
- 监管、通信、应急形成闭环。

## 11. 常见误区

### 11.1 误区：AIS 是雷达

AIS 不是雷达。AIS 依赖目标主动发射，雷达依赖电磁波反射。AIS 能给出身份信息，雷达能发现不发 AIS 的目标。两者应互补。

### 11.2 误区：接收到 AIS 就能自由转卖数据

不一定。接收许可、数据来源、信息传播、商业授权、数据安全和敏感目标都需要单独评估。

### 11.3 误区：DSC 可以替代 AIS

不能。DSC 是呼叫与遇险报警机制，AIS 是连续态势广播系统。

### 11.4 误区：AIS 可以替代 DSC 遇险报警

不能。AIS 可辅助搜救，但 DSC 是 GMDSS 中的正式数字呼叫/报警机制之一。

### 11.5 误区：买到 AIS 发射设备就能在陆地发射

不能。设备型号核准、频率许可、台站执照、海事管理要求是不同层级的问题。

## 16. 一页总结

AIS 是“自动广播态势系统”，重点在船舶身份、位置、轨迹和监管数据。DSC 是“数字呼叫与遇险报警系统”，重点在选择性呼叫、遇险报警和通信建立。工程上，AIS 更偏数据平台与态势感知，DSC 更偏安全通信与应急响应。销售上，AIS 讲“看见与管理”，DSC 讲“呼叫与救援”，两者结合可以形成海上交通感知、通信和应急闭环。
