# Secure FOTA - 安全固件升级方案

**版本**: 2.0  
**日期**: 2026-03-02  
**参考**: MCUboot + WireGuard 加密方案

---

## 📋 概述

本方案结合 MCUboot 的安全启动机制和 WireGuard 的 ChaCha20-Poly1305 加密算法，为资源受限的 MCU 提供完整的固件安全升级方案。

---

```
┌─────────────────────────────────────────────────┐
│              升级服务器                          │
│  - 固件签名 (ECDSA P-256)                       │
│  - 固件加密 (ChaCha20-Poly1305)                 │
│  - 密钥管理                                     │
└─────────────────┬───────────────────────────────┘
                  │ 加密固件包
                  ▼
┌─────────────────────────────────────────────────┐

│  │  - ECDSA 签名验证                        │    │

│  │  - Poly1305 完整性校验                  │    │
│  └─────────────────────────────────────────┘    │
│  ┌─────────────────────────────────────────┐    │
│  │  双 Bank Flash 管理                      │    │
│  │  - Slot 0: 运行区                        │    │
```

## 🔐 安全特性

### 1. 加密算法选择

| 算法 | 用途 | 密钥长度 | 说明 |
|------|------|----------|------|
| **ECDSA P-256** | 签名验证 | 256 位 | 非对称加密，验证固件来源 |
| **ChaCha20** | 固件加密 | 256 位 | 流加密，性能优于 AES |
| **Poly1305** | 完整性校验 | 256 位 | MAC 认证，防篡改 |

### 2. 密钥管理

```
根密钥对 (离线保存):
  - 私钥：HSM 安全存储，仅用于签名
  - 公钥：编译进 Bootloader

设备密钥对 (每设备唯一):
  - 私钥：设备安全存储区
  - 公钥：服务器注册

会话密钥 (每次升级生成):
  - ChaCha20 密钥：ECDH 密钥派生
  - Nonce：每次加密唯一
```

### 3. 启动流程

```
1. 设备上电
   ↓
2. Bootloader 启动
   ↓
3. 检查升级标志
   ├─ 有升级包 → 验证签名 → 解密固件 → 写入 Slot 0
   └─ 无升级包 → 验证 Slot 0 签名 → 跳转执行
   ↓
4. 应用程序运行
```

## 🔧 实现方案

### 1. Bootloader 内存布局

```
Flash Layout (Total 512KB):
┌─────────────────────────────────────┐
│ 0x00000 - 0x07FFF: Bootloader (32KB)│
│ 0x08000 - 0x0FFFF: Config (32KB)    │
│ 0x10000 - 0x3FFFF: Slot 0 (192KB)   │ ← 运行区
│ 0x40000 - 0x6FFFF: Slot 1 (192KB)   │ ← 升级区
│ 0x70000 - 0x7FFFF: Parameters (64KB)│
└─────────────────────────────────────┘
```

### 2. API 接口

```c
/* 初始化安全 FOTA */
int xy_fota_secure_init(xy_fota_secure_t *fota, 
                        const uint8_t *pub_key,
                        xy_fota_flash_ops_t *flash);

/* 验证并解密固件包 */
int xy_fota_secure_verify(xy_fota_secure_t *fota,
                          const uint8_t *fw_pkg,
                          uint32_t pkg_size);

/* 解密固件数据块 */
int xy_fota_secure_decrypt(xy_fota_secure_t *fota,
                           const uint8_t *in,
                           uint8_t *out,
                           uint32_t len);

/* 交换 Slot (升级完成后) */
int xy_fota_secure_swap(xy_fota_secure_t *fota);
```

### 3. 性能优化

| 优化项 | 说明 | 效果 |
|--------|------|------|
| **硬件加密引擎** | 使用 MCU 硬件 AES/SHA | 10x 速度提升 |
| **流式解密** | 边解密边写入 Flash | 减少 RAM 占用 |
| **双 Bank 切换** | 避免原地升级风险 | 提高可靠性 |
