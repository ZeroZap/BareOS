## 1. PID基础原理

### 1.1 什么是PID控制？

PID控制器是一种**闭环反馈控制器**，通过计算**设定值(Setpoint)\**与\**实际值(Feedback)\**之间的\**误差(Error)**，产生控制输出来消除误差。

```text
                    ┌─────────────────────────────────────────┐
                    │              PID控制器                   │
                    │  ┌─────┐   ┌─────┐   ┌─────┐           │
         误差 e(t)  │  │  P  │ + │  I  │ + │  D  │  输出 u(t)│
    ──────────────►│  │比例 │   │积分 │   │微分 │──────────►│
                    │  └─────┘   └─────┘   └─────┘           │
                    └─────────────────────────────────────────┘
                                     │
    ┌────────────────────────────────┼────────────────────────────┐
    │                                ▼                            │
    │   设定值 r(t)    ┌───┐     ┌───────┐     ┌──────┐          │
    │   ───────────►──►│ - │────►│  PID  │────►│ 被控 │────┬────►│ 输出 y(t)
    │                  └───┘     │控制器 │     │对象 │    │     │
    │                    ▲       └───────┘     └──────┘    │     │
    │                    │                                  │     │
    │                    └──────────────────────────────────┘     │
    │                              反馈                            │
    └─────────────────────────────────────────────────────────────┘
```

### 1.2 三个基本分量

| 分量  |        名称         |       作用       |         特点         |
| :---: | :-----------------: | :--------------: | :------------------: |
| **P** | 比例 (Proportional) | 快速响应当前误差 |      有稳态误差      |
| **I** |   积分 (Integral)   |   消除稳态误差   |   响应慢，可能振荡   |
| **D** |  微分 (Derivative)  | 预测误差变化趋势 | 抑制超调，对噪声敏感 |

### 1.3 连续域PID公式



u(t)=Kp⋅e(t)+Ki⋅∫0te(τ)dτ+Kd⋅de(t)dt*u*(*t*)=*K**p*⋅*e*(*t*)+*K**i*⋅∫0*t**e*(*τ*)*d**τ*+*K**d*⋅*d**t**d**e*(*t*)



其中：

- u(t)*u*(*t*) — 控制器输出
- e(t)=r(t)−y(t)*e*(*t*)=*r*(*t*)−*y*(*t*) — 误差（设定值 - 反馈值）
- Kp*K**p* — 比例系数
- Ki*K**i* — 积分系数
- Kd*K**d* — 微分系数

### 1.4 离散化

在数字系统中，需要将连续公式离散化：

|      连续域      |               离散域               |      说明       |
| :--------------: | :--------------------------------: | :-------------: |
|   e(t)*e*(*t*)   |            e(k)*e*(*k*)            | 第k次采样的误差 |
| ∫e⋅dt∫*e*⋅*d**t* |        ∑e(k)⋅T∑*e*(*k*)⋅*T*        |   矩形积分法    |
| dedt*d**t**d**e* | e(k)−e(k−1)T*T**e*(*k*)−*e*(*k*−1) |   后向差分法    |

其中 T*T* 为采样周期。

## 2. 位置式PID

### 2.1 原理

位置式PID直接计算控制器的**绝对输出值**，输出与执行机构的实际位置（或状态）直接对应。

### 2.2 数学公式



u(k)=Kp⋅e(k)+Ki⋅∑j=0ke(j)⋅T+Kd⋅e(k)−e(k−1)T*u*(*k*)=*K**p*⋅*e*(*k*)+*K**i*⋅∑*j*=0*k**e*(*j*)⋅*T*+*K**d*⋅*T**e*(*k*)−*e*(*k*−1)



简化形式（将T吸收到系数中）：



u(k)=Kp⋅e(k)+Ki⋅∑j=0ke(j)+Kd⋅[e(k)−e(k−1)]*u*(*k*)=*K**p*⋅*e*(*k*)+*K**i*⋅∑*j*=0*k**e*(*j*)+*K**d*⋅[*e*(*k*)−*e*(*k*−1)]



### 2.3 计算流程图

```text
开始
  │
  ▼
┌─────────────────────────┐
│ 读取设定值 r(k)         │
│ 读取反馈值 y(k)         │
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────┐
│ 计算误差                │
│ e(k) = r(k) - y(k)      │
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────┐
│ 计算比例项              │
│ P = Kp × e(k)           │
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────┐
│ 更新积分累积            │
│ integral += e(k) × T    │
│ 积分限幅                │
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────┐
│ 计算积分项              │
│ I = Ki × integral       │
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────┐
│ 计算微分项              │
│ D = Kd × (e(k)-e(k-1))/T│
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────┐
│ 计算输出                │
│ u(k) = P + I + D        │
│ 输出限幅                │
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────┐
│ 保存历史值              │
│ e(k-1) = e(k)           │
└───────────┬─────────────┘
            │
            ▼
          结束
```

### 2.4 代码实现

```c
pid_real_t pid_calc_positional(pid_controller_t *pid)
{
    pid_real_t p_term, i_term, d_term;
    
    /* ========== 比例项 ========== */
    // P = Kp × e(k)
    p_term = PID_REAL_MUL(pid->params.kp, pid->error);
    
    /* ========== 积分项 ========== */
    // integral += e(k) × T
    pid->integral += PID_REAL_MUL(pid->error, pid->dt);
    
    // 积分限幅，防止积分饱和
    pid->integral = pid_clamp(pid->integral, 
                              pid->integral_min, 
                              pid->integral_max);
    
    // I = Ki × integral
    i_term = PID_REAL_MUL(pid->params.ki, pid->integral);
    
    /* ========== 微分项 ========== */
    // D = Kd × [e(k) - e(k-1)] / T
    pid_real_t diff = PID_REAL_DIV(pid->error - pid->error_prev, pid->dt);
    d_term = PID_REAL_MUL(pid->params.kd, diff);
    
    /* ========== 总输出 ========== */
    // u(k) = P + I + D
    pid->output = p_term + i_term + d_term;
    
    return pid->output;
}
```

## 3. 增量式PID

### 3.1 原理

增量式PID计算的是控制量的**增量**（变化量），而非绝对值。当前输出 = 上次输出 + 本次增量。

### 3.2 数学推导

位置式公式：

u(k)=Kp⋅e(k)+Ki⋅∑j=0ke(j)+Kd⋅[e(k)−e(k−1)]*u*(*k*)=*K**p*⋅*e*(*k*)+*K**i*⋅∑*j*=0*k**e*(*j*)+*K**d*⋅[*e*(*k*)−*e*(*k*−1)]



上一时刻：

u(k−1)=Kp⋅e(k−1)+Ki⋅∑j=0k−1e(j)+Kd⋅[e(k−1)−e(k−2)]*u*(*k*−1)=*K**p*⋅*e*(*k*−1)+*K**i*⋅∑*j*=0*k*−1*e*(*j*)+*K**d*⋅[*e*(*k*−1)−*e*(*k*−2)]



两式相减得增量：

Δu(k)=u(k)−u(k−1)Δ*u*(*k*)=*u*(*k*)−*u*(*k*−1)





Δu(k)=Kp⋅[e(k)−e(k−1)]+Ki⋅e(k)+Kd⋅[e(k)−2e(k−1)+e(k−2)]Δ*u*(*k*)=*K**p*⋅[*e*(*k*)−*e*(*k*−1)]+*K**i*⋅*e*(*k*)+*K**d*⋅[*e*(*k*)−2*e*(*k*−1)+*e*(*k*−2)]



最终输出：

u(k)=u(k−1)+Δu(k)*u*(*k*)=*u*(*k*−1)+Δ*u*(*k*)

开始

### 3.4 代码实现

```c
pid_real_t pid_calc_incremental(pid_controller_t *pid)
{
    pid_real_t delta_p, delta_i, delta_d;
    pid_real_t delta_output;
    
    /* ========== 增量比例项 ========== */
    // ΔP = Kp × [e(k) - e(k-1)]
    delta_p = PID_REAL_MUL(pid->params.kp, 
                           pid->error - pid->error_prev);
    
    /* ========== 增量积分项 ========== */
    // ΔI = Ki × e(k) × T
    delta_i = PID_REAL_MUL(pid->params.ki, 
                           PID_REAL_MUL(pid->error, pid->dt));
    
    /* ========== 增量微分项 ========== */
    // ΔD = Kd × [e(k) - 2×e(k-1) + e(k-2)] / T
    pid_real_t diff_term = pid->error 
                         - pid->error_prev 
                         - pid->error_prev 
                         + pid->error_prev2;
    delta_d = PID_REAL_MUL(pid->params.kd, 
                           PID_REAL_DIV(diff_term, pid->dt));
    
    /* ========== 计算增量 ========== */
    delta_output = delta_p + delta_i + delta_d;
    
    /* ========== 累加到输出 ========== */
    // u(k) = u(k-1) + Δu(k)
    pid->output = pid->output_prev + delta_output;
    
    return pid->output;
}
```

## 4. 积分分离PID

### 4.1 原理

当误差较大时，积分作用会导致积分项过大，造成**超调**和**振荡**。积分分离的思想是：

- **误差大时**：禁用积分，只用PD控制，快速接近目标
- **误差小时**：启用积分，消除稳态误差

```text
        误差 e(k)
            │
            ▼
    ┌───────────────┐
    │ |e(k)| > 阈值? │
    └───────┬───────┘
            │
      ┌─────┴─────┐
      │ YES       │ NO
      ▼           ▼
  ┌───────┐   ┌───────┐
  │ PD控制 │   │PID控制│
  │ I项=0  │   │ 正常  │
  └───────┘   └───────┘
```

### 4.2 数学公式

定义积分分离系数 β*β*：



β={0,∣e(k)∣>ε1,∣e(k)∣≤ε*β*={0,1,∣*e*(*k*)∣>*ε*∣*e*(*k*)∣≤*ε*



其中 ε*ε* 为积分分离阈值。

输出公式：

u(k)=Kp⋅e(k)+β⋅Ki⋅∑e(k)+Kd⋅[e(k)−e(k−1)]*u*(*k*)=*K**p*⋅*e*(*k*)+*β*⋅*K**i*⋅∑*e*(*k*)+*K**d*⋅[*e*(*k*)−*e*(*k*−1)]



### 4.3 改进：渐变式积分分离

硬切换可能导致输出不连续，可采用渐变系数：



β={0,∣e(k)∣>ε2ε2−∣e(k)∣ε2−ε1,ε1<∣e(k)∣≤ε21,∣e(k)∣≤ε1*β*=⎩⎨⎧0,*ε*2−*ε*1*ε*2−∣*e*(*k*)∣,1,∣*e*(*k*)∣>*ε*2*ε*1<∣*e*(*k*)∣≤*ε*2∣*e*(*k*)∣≤*ε*1



```text
β (积分系数)
│
1├─────────────────────┐
│                      │
│                      │
│                      ╲╲
│                        ╲
0├────────────────────────┴─────────────
  0        ε₁         ε₂        |e(k)|
           │          │
           │          └── 完全禁用积分
           └── 完全启用积分
```

                                  pid->integral_min, 
                                  pid->integral_max);
    }
    // 注意：即使不累积，也使用已有的积分值
    i_term = PID_REAL_MUL(pid->params
    
    /* ========== 微分项 ========== */

    d_term = PID_REAL_MUL(pid->params.kd, diff);
    
    /* ========== 总输出 ========== */
    pid->output = p_term + i_term + d_term;

## 5. 抗积分饱和PID

### 5.1 积分饱和问题

当执行器达到物理极限（如电机满功率、阀门全开）时，即使控制器输出继续增加，实际作用也不会改变。但积分项会继续累积，导致：

1. **积分项过大**（积分饱和）
2. **误差反向后**，需要很长时间才能"消化"掉累积的积分
3. **造成严重超调**

```text
输出
 │
 │  输出上限 ─────────────────────────────────────
 │           │xxxxxxxxxxxxxxx│
 │           │ 积分继续累积  │  ← 积分饱和区
 │           │ 但输出被限幅  │
 │           │xxxxxxxxxxxxxxx│
 │        ╱──┘               └──╲
 │      ╱                        ╲
 │    ╱                            ╲  ← 超调
 │  ╱                                ╲
 ├─╱──────────────────────────────────╲────────► 时间
 │                                      ╲
 │                                        ╲____ 目标值
```

### 5.2 Back-calculation 方法

这是最常用的抗积分饱和方法，原理是：

**当输出饱和时，通过反馈修正积分项，使其不会过度累积。**

```text
                                    ┌─────────┐
                                    │  限幅   │
         ┌───────────────────────►──┤         ├──────► u(k) 输出
         │                          │ [min,max]│
         │                          └────┬────┘
         │                               │
         │    ┌──────────────────────────┘
         │    │ 饱和误差 = u_限幅后 - u_限幅前
         │    ▼
    ┌────┴────┐      ┌─────┐
    │   PID   │◄─────┤ ×Kb │◄──── 饱和误差反馈
    │  计算   │      └─────┘
    └─────────┘
         ▲
         │
      e(k) 误差
```

### 5.3 数学公式

标准PID输出（未限幅）：

uunsat(k)=Kp⋅e(k)+Ki⋅I(k)+Kd⋅D(k)*u**u**n**s**a**t*(*k*)=*K**p*⋅*e*(*k*)+*K**i*⋅*I*(*k*)+*K**d*⋅*D*(*k*)



限幅后输出：

u(k)=clamp(uunsat(k),umin,umax)*u*(*k*)=clamp(*u**u**n**s**a**t*(*k*),*u**min*,*u**ma**x*)



饱和误差：

esat(k)=u(k)−uunsat(k)*e**s**a**t*(*k*)=*u*(*k*)−*u**u**n**s**a**t*(*k*)



积分更新（带抗饱和）：

I(k+1)=I(k)+[e(k)+Kb⋅esat(k)]⋅T*I*(*k*+1)=*I*(*k*)+[*e*(*k*)+*K**b*⋅*e**s**a**t*(*k*)]⋅*T*



其中 Kb*K**b* 为抗饱和反馈系数，通常取 Kb=1/Ki*K**b*=1/*K**i* 或根据经验调整。

### 5.4 工作原理图解

```text
情况1: 输出未饱和
─────────────────────────────────────────────────
u_unsat = 50,  u_max = 100
u = clamp(50, -100, 100) = 50
e_sat = 50 - 50 = 0  ← 无饱和误差
积分正常累积

情况2: 输出饱和
─────────────────────────────────────────────────
u_unsat = 150,  u_max = 100
u = clamp(150, -100, 100) = 100
e_sat = 100 - 150 = -50  ← 负的饱和误差

积分更新: I += (e + Kb × (-50)) × T
                    ↑
                负反馈，减少积分累积
```

### 5.5 代码实现

```c
pid_real_t pid_calc_anti_windup(pid_controller_t *pid)
{
    pid_real_t p_term, i_term, d_term;
    pid_real_t output_unsat;
    pid_real_t saturation_error;
    
    /* ========== 比例项 ========== */
    p_term = PID_REAL_MUL(pid->params.kp, pid->error);
    
    /* ========== 积分项 (使用当前积分值) ========== */
    i_term = PID_REAL_MUL(pid->params.ki, pid->integral);
    
    /* ========== 微分项 ========== */
    pid_real_t diff = PID_REAL_DIV(pid->error - pid->error_prev, pid->dt);
    d_term = PID_REAL_MUL(pid->params.kd, diff);
    
    /* ========== 计算未限幅输出 ========== */
    output_unsat = p_term + i_term + d_term;
    
    /* ========== 限幅 ========== */
    pid->output = pid_clamp(output_unsat, 
                            pid->output_min, 
                            pid->output_max);
    
    /* ========== 计算饱和误差 ========== */
    // e_sat = u_限幅后 - u_限幅前
    saturation_error = pid->output - output_unsat;
    
    // 更新饱和标志
    pid->saturated = (saturation_error != PID_ZERO);
    
    /* ========== 抗饱和积分更新 ========== */
    // I(k+1) = I(k) + [e(k) + Kb × e_sat] × T
    pid_real_t integral_input = pid->error + 
                                PID_REAL_MUL(pid->kb, saturation_error);
    pid->integral += PID_REAL_MUL(integral_input, pid->dt);
    
    // 积分限幅（双重保护）
    pid->integral = pid_clamp(pid->integral, 
                              pid->integral_min, 
                              pid->integral_max);
    
    return pid->output;
}
```

## 6. 设定值加权PID

### 6.1 问题背景

标准PID中，设定值的突变会直接影响误差，导致：

- **比例项突变** → 输出跳变
- **微分项冲击** → 微分踢（Derivative Kick）

```text
标准PID响应设定值阶跃
─────────────────────────────────────────────────
设定值 r(t)
    │
100 ├─────────────────┐
    │                 │
    │                 │
  0 ├─────────────────┴─────────────────────► t
    
误差 e(t) = r(t) - y(t)
    │
100 ├─────────────────┐  ← 误差突变
    │                 │
    │                 ╲
    │                  ╲
  0 ├───────────────────╲───────────────────► t

微分项 de/dt
    │
    │  ← 微分冲击（理论上无穷大）
  ∞ ├─┐
    │ │
    │ │
  0 ├─┴─────────────────────────────────────► t
```

### 6.2 解决方案

设定值加权PID对三个分量使用不同的误差计算方式：

| 分量 |               标准PID                |               设定值加权PID                |
| :--: | :----------------------------------: | :----------------------------------------: |
| P项  |       Kp⋅(r−y)*K**p*⋅(*r*−*y*)       |       Kp⋅(b⋅r−y)*K**p*⋅(*b*⋅*r*−*y*)       |
| I项  |  Ki⋅∫(r−y)dt*K**i*⋅∫(*r*−*y*)*d**t*  |     Ki⋅∫(r−y)dt*K**i*⋅∫(*r*−*y*)*d**t*     |
| D项  | Kd⋅d(r−y)dt*K**d*⋅*d**t**d*(*r*−*y*) | Kd⋅d(c⋅r−y)dt*K**d*⋅*d**t**d*(*c*⋅*r*−*y*) |

其中：

- b*b* — 比例项设定值权重 (0~1)
- c*c* — 微分项设定值权重 (0~1)
- 积分项始终使用完整误差，确保稳态精度

### 6.3 权重系数的作用

```text
b = 1, c = 1 (标准PID)
─────────────────────────────────────────────────
    │     ╭──────────────────────────
    │   ╭─╯
    │ ╭─╯    超调
    │╭╯
────┼╯─────────────────────────────────────────►
    │

b = 0.7, c = 0 (设定值加权)
─────────────────────────────────────────────────
    │         ╭──────────────────────
    │       ╭─╯
    │     ╭─╯   减少超调，响应更平滑
    │   ╭─╯
────┼──╯───────────────────────────────────────►
    │

b = 0, c = 0 (I-PD控制器)
─────────────────────────────────────────────────
    │              ╭─────────────────
    │           ╭──╯
    │        ╭──╯   无超调，但响应慢
    │     ╭──╯
────┼────╯─────────────────────────────────────►
    │
```

### 6.4 数学公式



u(k)=Kp⋅(b⋅r−y)+Ki⋅∑(r−y)+Kd⋅d(c⋅r−y)dt*u*(*k*)=*K**p*⋅(*b*⋅*r*−*y*)+*K**i*⋅∑(*r*−*y*)+*K**d*⋅*d**t**d*(*c*⋅*r*−*y*)



展开：

- **P项**：P=Kp⋅(b⋅r(k)−y(k))*P*=*K**p*⋅(*b*⋅*r*(*k*)−*y*(*k*))
- **I项**：I=Ki⋅∑[r(k)−y(k)]⋅T*I*=*K**i*⋅∑[*r*(*k*)−*y*(*k*)]⋅*T*
- **D项**：D=Kd⋅[c⋅r(k)−y(k)]−[c⋅r(k−1)−y(k−1)]T*D*=*K**d*⋅*T*[*c*⋅*r*(*k*)−*y*(*k*)]−[*c*⋅*r*(*k*−1)−*y*(*k*−1)]

### 6.5 代码实现

```c
pid_real_t pid_calc_setpoint_weight(pid_controller_t *pid)
{
    pid_real_t p_term, i_term, d_term;
    pid_real_t p_error, d_error;
    static pid_real_t d_error_prev = 0;
    
    /* ========== 计算加权误差 ========== */
    // 比例项误差: b×r - y
    p_error = PID_REAL_MUL(pid->weight_b, pid->setpoint) - pid->feedback;
    
    // 微分项误差: c×r - y
    d_error = PID_REAL_MUL(pid->weight_c, pid->setpoint) - pid->feedback;
    
    /* ========== 比例项 ========== */
    // P = Kp × (b×r - y)
    p_term = PID_REAL_MUL(pid->params.kp, p_error);
    
    /* ========== 积分项 (使用完整误差) ========== */
    // I = Ki × Σ(r - y)
    // 注意：积分使用完整误差 pid->error = r - y
    pid->integral += PID_REAL_MUL(pid->error, pid->dt);
    pid->integral = pid_clamp(pid->integral, 
                              pid->integral_min, 
                              pid->integral_max);
    i_term = PID_REAL_MUL(pid->params.ki, pid->integral);
    
    /* ========== 微分项 ========== */
    // D = Kd × d(c×r - y)/dt
    pid_real_t diff = PID_REAL_DIV(d_error - d_error_prev, pid->dt);
    d_error_prev = d_error;  // 保存历史值
    d_term = PID_REAL_MUL(pid->params.kd, diff);
    
    /* ========== 总输出 ========== */
    pid->output = p_term + i_term + d_term;
    
    return pid->output;
}
```

|  减少超调  | 0.7  | 0.0  | 平滑响应 |

## 7. Pessen积分规则PID

### 7.1 原理

Pessen积分规则是一种**条件积分**策略，核心思想：

**只有当误差穿越零点（符号改变）时，才累积积分。**

这样可以：

- 避免同方向误差持续累积
- 减少超调
- 加快收敛

```text
误差 e(t)
    │
    │    ╭╮
    │   ╱  ╲
  + ├──╱────╲─────────────────────────────────
    │ ╱      ╲    ╭╮
    │╱        ╲  ╱  ╲
  0 ├──────────╲╱────╲────────────────────────
    │           ╳     ╲  ╱
    │          ↑ ╲    ╲╱
  - ├──────────┼──╲────────────────────────────
    │          │
    │     符号改变点
    │     此时累积积分
```

### 7.2 数学公式

定义积分使能条件：



integrate={true,e(k)⋅e(k−1)<0 (符号改变)true,∣e(k)∣<ε (误差很小)false,其他情况integrate=⎩⎨⎧true,true,false,*e*(*k*)⋅*e*(*k*−1)<0 (符号改变)∣*e*(*k*)∣<*ε* (误差很小)其他情况



积分更新：

I(k)={I(k−1)+e(k)⋅T,integrate = trueI(k−1),integrate = false*I*(*k*)={*I*(*k*−1)+*e*(*k*)⋅*T*,*I*(*k*−1),integrate = trueintegrate = false



### 7.3 代码实现

```c
pid_real_t pid_calc_pessen_integral(pid_controller_t *pid)
{
    pid_real_t p_term, i_term, d_term;
    bool sign_changed;
    bool error_small;
    bool do_integrate;
    
    /* ========== 检测积分条件 ========== */
    // 条件1: 误差符号改变
    sign_changed = ((pid->error > PID_ZERO && pid->error_prev < PID_ZERO) ||
                    (pid->error < PID_ZERO && pid->error_prev > PID_ZERO));
    
    // 条件2: 误差很小
    error_small = (PID_REAL_ABS(pid->error) < pid->integral_sep_thresh);
    
    // 满足任一条件则累积积分
    do_integrate = sign_changed || error_small;
    
    /* ========== 比例项 ========== */
    p_term = PID_REAL_MUL(pid->params.kp, pid->error);
    
    /* ========== 积分项 (条件累积) ========== */
    if (do_integrate) {
        pid->integral += PID_REAL_MUL(pid->error, pid->dt);
        pid->integral = pid_clamp(pid->integral, 
                                  pid->integral_min, 
                                  pid->integral_max);
    }
    i_term = PID_REAL_MUL(pid->params.ki, pid->integral);
    
    /* ========== 微分项 ========== */
    pid_real_t diff = PID_REAL_DIV(pid->error - pid->error_prev, pid->dt);
    d_term = PID_REAL_MUL(pid->params.kd, diff);
    
    /* ========== 总输出 ========== */
    pid->output = p_term + i_term + d_term;
    
    return pid->output;
}
```

## 📝 总结

1. **位置式PID** — 最基础，输出绝对值，适合大多数场景
2. **增量式PID** — 输出增量，无积分饱和，适合增量执行器
3. **积分分离PID** — 大误差时禁用积分，减少超调
4. **抗积分饱和PID** — 通过反馈修正积分，防止饱和
5. **设定值加权PID** — 分离跟踪和抗扰性能，减少设定值冲击
6. **Pessen积分PID** — 条件积分，减少振荡

选择合适的算法需要考虑：

- 被控对象特性
- 执行器类型和限制
- 性能要求（响应速度、超调、精度）
- 实现复杂度
