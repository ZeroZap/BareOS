# BareOS 架构设计

> 版本: 3.0 | 日期: 2026-05-14
> 目标产品: 船舶求救定位模块 (PLB)

---

## 一、设计约束

| 约束 | 说明 |
|------|------|
| **纯裸机** | 不使用 RTOS，不依赖 FreeRTOS / RT-Thread |
| **通信协议** | 全部通过 AT 指令 over UART 与外部模块交互 |
| **处理器** | Cortex-M 系列（STM32/WCH/HC32），无 MMU |
| **内存** | RAM ≤ 128KB，ROM ≤ 512KB |
| **电源** | 电池供电，需支持 tickless 低功耗睡眠 |

## 三、执行模型：裸机前后台

### 3.1 主循环

```c
void main(void)
{
    bsp_init();      /* 外设初始化：UART、GPIO、SysTick */
    app_init();      /* 注册所有任务 */

    while (1)
    {
        /* ① 驱动各 AT 对象（非阻塞轮询，内部运行工作队列）*/
        at_obj_process(g_at_4g);
        at_obj_process(g_at_gnss);
        at_obj_process(g_at_sat);  /* 卫星模块 */

        /* ② 运行各 Protothread 任务 */
        PT_SCHEDULE(task_gnss(&s_pt_gnss));
        PT_SCHEDULE(task_comm(&s_pt_comm));
        PT_SCHEDULE(task_distress(&s_pt_distress));

        /* ③ 低功耗：所有任务空闲时进 Stop 模式 */
        if (all_tasks_idle()) {
            bsp_enter_stop_until_uart_or_timer();
        }
    }
}
```

### 3.2 前台（ISR）职责

ISR **只做以下三件事**：

1. **UART RX ISR** — 将收到的字节写入 `at_obj` 的 RX 环形缓冲区（由 AT-Command-V2 的 `read` 回调管理）
2. **SysTick ISR** — 递增 ms 计数器（驱动 `at_get_ms()`、`etimer`、`xy_timer`）
3. **GPIO ISR（按键）** — 置标志位，主循环处理

## 四、Protothread 并发模型

无栈协程，用 Duff's Device 技术实现挂起/恢复，无需独立栈帧。

```c
PT_THREAD(task_comm(struct pt *pt))
{
    static struct pt   sub;
    static at_context_t ctx;
    static char         buf[64];

    PT_BEGIN(pt);

    /* 步骤 1：握手 */
    at_context_init(&ctx, buf, sizeof(buf));
    at_send_singlline(g_at_4g,
        &(at_attr_t){ .timeout=1000, .ctx=&ctx, .cb=NULL },
        "AT");
    PT_WAIT_UNTIL(pt, at_work_is_finish(&ctx));
    if (at_work_get_result(&ctx) != AT_RESP_OK) { PT_EXIT(pt); }

    /* 步骤 2：查信号质量 */
    at_context_init(&ctx, buf, sizeof(buf));
    at_exec_cmd(g_at_4g,
        &(at_attr_t){ .timeout=2000, .prefix="+CSQ", .ctx=&ctx },
        "AT+CSQ");
    PT_WAIT_UNTIL(pt, at_work_is_finish(&ctx));

    PT_END(pt);
}
```

**PT_WAIT_UNTIL** 是核心原语 — 每次主循环都会调用此函数，条件不满足则立即返回让出 CPU，条件满足时从断点继续，实现协作式"等待"而不阻塞主循环。

## 五、AT-Command-V2 框架

来自 `components-ref/net/at/AT-Command-V2/`，**无 RTOS 依赖**，纯回调 + 轮询驱动。

### 5.1 架构

```
主循环每轮调用 at_obj_process()
    │
    ├── 从 adapter.read() 读取 UART 数据（非阻塞）
    ├── 匹配 URC 前缀 → 触发 urc_item_t.handler()
    └── 驱动工作队列：SEND → 等待响应 → 超时/OK/ERROR → 回调 cb
```

### 5.2 适配器（adapter）接口

每个 UART 外设对应一个 `at_obj_t`，通过 `at_adapter_t` 绑定收发函数：

```c
static const at_adapter_t adapter_4g = {
    .write        = uart1_write,      /* 非阻塞发送 */
    .read         = uart1_read,       /* 非阻塞接收，从 ISR 环形缓冲读 */
    .error        = NULL,
    .debug        = at_debug_print,   /* 调试输出，NULL=禁用 */
    .lock         = NULL,             /* 裸机不需要锁 */
    .unlock       = NULL,
    .recv_bufsize = 256,
    .urc_bufsize  = 128,
};
```

### 5.3 URC（主动上报）处理

```c
static const urc_item_t urcs_4g[] = {
    { .prefix = "+QMTRECV",  .endmark = '\n', .handler = on_mqtt_recv },
    { .prefix = "+QMTSTAT",  .endmark = '\n', .handler = on_mqtt_stat },
    { .prefix = "+CGREG",    .endmark = '\n', .handler = on_net_reg   },
};
at_obj_set_urc(g_at_4g, urcs_4g, ARRAY_SIZE(urcs_4g));
```

### 5.4 at_port.c 裸机适配

```c
/* components/at/src/at_port.c */
void *at_malloc(unsigned int n) { return malloc(n);  }   /* 或 xy_mem 池 */
void  at_free(void *p)          { free(p);            }
unsigned int at_get_ms(void)    { return g_sys_tick_ms; } /* SysTick 计数 */
```

## 六、状态机：`sm/`

处理持续性状态监控，与 Protothread 分工：

| 场景 | 使用 |
|------|------|
| 线性多步序列（A→B→C，且有等待） | Protothread |
| 事件驱动状态跳转（任意方向）| 状态机 |
| 复杂协议（超时重试 + 状态持续）| 状态机 + Protothread |

PLB 顶层状态机示例：

```
IDLE ──按键──→ ACQUIRING_FIX ──定位成功──→ TRANSMITTING ──发完──→ IDLE
  ↑                 │ 超时 60s                  │ 重发间隔 50s        │
  └─────────────────┴───────────────────────────┴────────────────────┘
```

## 七、硬件模块与 UART 映射

| 模块 | UART | AT 对象 | 用途 |
|------|------|---------|------|
| 4G 模块（EC200U/Air780E） | UART1 | `g_at_4g` | 位置上报、AIS 接入 |
| GNSS 模块（AT6558/L76K） | UART2 | `g_at_gnss` | NMEA 定位句子 |
| 卫星通信模块（Globalstar/BeiDou RDSS） | UART3 | `g_at_sat` | 求救报文发送 |
| 调试串口 | UART0 | — | 日志输出 |

> GNSS 模块也可以工作在 NMEA 被动模式（只接收 $GPRMC 等句子，不需要 AT 交互），
> 此时由 `components/gnss/` 的 NMEA 解析器直接处理 UART2 的原始数据流。

## 九、关键数据流：求救发送

```
按键按下 (ISR)
    │ 置 g_sos_request = 1
    ▼
task_distress (PT)
    │ SM: IDLE → ACQUIRING_FIX
    │   PT_WAIT_UNTIL(pt, gnss_has_valid_fix() || etimer_expired(&fix_timeout))
    │
    ├─ 定位成功 → SM: ACQUIRING_FIX → TRANSMITTING
    │   构造求救报文（MMSI + lat/lon + 时间戳）
    │   AT: AT+QISEND → 通过 4G 发送
    │   AT: 卫星模块发送求救报文
    │   每 50s 重发一次（etimer）
    │
    └─ 超时/取消 → SM: TRANSMITTING → IDLE
```

## 十、低功耗策略

1. **tickless sleep** — 所有 etimer 均无待期时，关闭 SysTick，进入 Stop 模式
2. **UART 唤醒** — 任意 UART 收到字节触发中断，退出 Stop
3. **定时唤醒** — 用 LPTimer（低功耗定时器）在 Stop 模式下保持计时，到期唤醒主循环
4. **模块控制** — GNSS/4G 空闲时通过 GPIO 断电（`AT+QPOWD` 软关闭后拉低使能脚）
